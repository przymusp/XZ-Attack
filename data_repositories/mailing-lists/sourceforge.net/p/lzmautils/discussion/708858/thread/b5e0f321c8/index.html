  • Lasse Collin - 2020-02-05

    On my system lld says this:

    lld is a generic driver.
    Invoke ld.lld (Unix), ld64.lld (macOS), lld-link (Windows), wasm-ld (WebAssembly) instead

    Which is a bit confusing but sounds like that perhaps this lld shouldn't be used directly.

    There are messages on the web that GNU Libtool lacks direct support for LLD. My impression is that Libtool has a severe lack of resources (no active developers). ld.lld works around the lack of direct Libtool support by adding the string "GNU" in the ld.lld --version output which makes Libtool treat it like GNU ld.
