    • Stuart Shelton - 2008-05-08

      Rightly or wrongly, on IRIX stdint.h is a C99 header - if you try to include it in C++ (or indeed C89) code, then it #errors out.  Therefore, you *cannot* include stdint.h

      The reason that HAVE_STDINT_H is defined is that the configure script is run with 'cc' - the C compiler which defaults to C99 mode and therefore thinks that stdint.h is okay to include.

      (You appear to have misread the patch - it specifically *excludes* stdint.h where __sgi is defined...)

      It's possible that this could be fixed by setting an environment variable in order to tell configure that HAVE_STDINT_H is false on IRIX.

      I can't actually remember what problem I was trying to fix with the SIZE_MAX change, but I'll recompile with the section moved downwards, and see what happens.

      Finally, the ReverseBitTree* functions do need to be marked as 'inline' as you previously discovered.
