  • Lasse Collin - 2009-09-25

    It was incorrect use of the &quot;restrict&quot; keyword in lzma_vli_encode() and lzma_vli_decode(), which handle variable-length integers. I simply removed &quot;restrict&quot; from those places.

  • Igor Pavlov - 2009-09-25

    But you use &quot;restrict&quot; in other places.
    Why it was incorrect in lzma_vli_encode?

     
    If you would like to refer to this comment somewhere else in this project, copy and paste the following link:

  • Lasse Collin - 2009-09-25

    Normally the data pointed by a restrict pointer is available only through that pointer. But the VLI functions check if the pointer is NULL, and if it is, set it to point to a variable on stack. Later the code compares if the pointer is pointing to that local variable (to know if it was originally NULL). These things aren't allowed for restrict pointers.
