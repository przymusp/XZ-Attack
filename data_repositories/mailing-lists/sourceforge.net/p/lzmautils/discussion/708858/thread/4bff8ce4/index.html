  • Lasse Collin - 2011-03-30

    XZ Utils cannot decompress .7z files. You need to tell 7-Zip to create .xz files if you want to decompress the files with XZ Utils. You need 7-Zip 9 for .xz support; 4.65 doesn't support .xz.

    You aren't setting the extreme flag in your code. You should bitwise-or LZMA_PRESET_EXTREME instead of using 1.

    You cannot get faster compression with XZ Utils than with 7-Zip.

  • Lasse Collin - 2011-03-31

    That 0xFD is the first byte in the .xz format. So if you compress into the .xz format, then what you see is normal.

    A few clarifications:

    .7z, .lzma, and .xz are file formats. 7-Zip 9.20 supports them all. XZ Utils supports .lzma and .xz. LZMA Utils (legacy software) supports only .lzma.

    LZMA and LZMA2 are compression methods. Normally they are used inside some file format; it is unusual to store raw LZMA or LZMA2 data without wrapping it into some file format. .7z supports both LZMA and LZMA2, .xz supports LZMA2, and .lzma supports only LZMA. Other file formats may internally use raw LZMA, raw LZMA2, .xz, or .lzma (but .lzma shouldn't be used for new applications).

    You wrote lzma920 in the topic. That refers to LZMA SDK 9.20 (from 7-Zip.org). The code you pasted uses liblzma from XZ Utils (tukaani.org). Both LZMA SDK and liblzma have C API, but the APIs are different.

    LZMA SDK has also C++, Java, and C# APIs. Only the C++ code supports compressing into the .7z format. LZMA SDK C API supports compressing and decompressing .lzma and .xz files and raw streams and decompressing .7z files.

  • Lasse Collin - 2011-04-01

    Both .7z and .xz can store LZMA2 compressed data. If you use 7-Zip or LZMA SDK, the same LZMA2 code is used for both file formats. With the same compression options, the sizes of .7z and .xz files should only differ by a couple of bytes. If you use liblzma from XZ Utils like you did in your example, the size difference might be a tiny bit bigger due to differences in implementations and default compression options.

    To get the very best compression, you may want to tweak LZMA2 parameters. It depends on the type of data you are compressing. For example, with plain text files it tends to help a little if you set pb=0 (default is pb=2).

    Note that if you compress x86 executable files, 7-Zip may create significantly smaller file when using the .7z format. This is because it can automatically apply BCJ2 filter which improves compression of x86 executables.
